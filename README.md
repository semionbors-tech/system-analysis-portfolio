# system-analysis-portfolio портфолио системный аналитик.

Проект 1: Проектирование архитектуры голосового ИИ-ассистента для автоматизации Аварийно-диспетчерской службы (АДС) управляющей компании ЖКХ
Роль: Системный аналитик (Проектировщик ИИ-интеграций)
Стек и стандарты: REST API, JSON, Speech-to-Text/Text-to-Speech (STT/TTS), LLM (RAG), BPMN, ГОСТ-ориентированная структура.

1. Бизнес-цель проекта (Business Goals): 1)Сократить нагрузку на живых операторов АДС на 70% за счет автоматизации обработки входящих звонков от жильцов. 2)Обеспечить мгновенный прием заявок по аварийным ситуациям и информирование по тарифам ЖКХ в режиме 24/7 без привлечения дополнительного штата диспетчеров.

2. Архитектура системы и интеграционные потоки (Data Flow)
Система представляет собой low-code платформу, связывающую через API четыре ключевых модуля:  

1.SIP-Телефония: Прием входящего аудиопотока. 

 2.Модуль STT/TTS (Yandex SpeechKit / SaluteSpeech): Конвертация «Голос \(\leftrightarrow \) Текст» с распознаванием естественных интонаций.

3.Оркестратор (LLM-мозг): Анализ контекста, извлечение сущностей (Entities), маршрутизация логики диалога.

4. CRM-система УК / База данных (Google Sheets/PostgreSQL): Фиксация заявок и хранение тарифов.


3. Функциональные требования и сценарии (Use Cases)Сценарий 1. Регистрация аварийной заявки (Авария/Сантехника)Предусловие: Жилец совершает звонок на горячую линию УК.Основной успешный сценарий:Модуль STT переводит речь жильца в текстовый лог.LLM-мозг извлекает ключевые сущности (Адрес, Категория проблемы, Уровень критичности).На основе извлеченных данных система формирует JSON-пакет и отправляет POST-запрос в CRM-систему УК.Модуль TTS озвучивает жильцу регламентное время прибытия бригады и номер заявки.

.Сценарий 2. Интеллектуальная фильтрация по тарифам (RAG-модуль)Логика работы: При запросе о стоимости услуг (например: «Почему за отопление так много?»), LLM  активирует фильтр по ключевому слову (Отопление).
Действие: Система запрашивает актуальный тариф Региональной энергетической комиссии (РЭК) для конкретного адреса, выдает точные цифры и автоматически инициирует отправку СМС-сообщения со ссылкой на официальный калькулятор начислений.

Сценарий 3. Обработка конфликтов и эскалация (Escalation Rule)Условие срабатывания: Детекция нецензурной лексики (по словарю стоп-слов) или превышение порога громкости аудиопотока (крик, паника) в течение более 3 секунд.

Алгоритм системы: Бот немедленно прерывает генерацию речи.Система отправляет SIP-команду на перевод звонка на живого оператора АДС.В CRM оператора выводится пуш-уведомление со статусом КРИТИЧЕСКИЙ / АГРЕССИЯ.На экран оператора выводится текстовый лог того, что клиент успел сказать роботу, и скрипт перехвата внимания: «Внимание! Назовите время прибытия (20 минут) и подтвердите адрес»
